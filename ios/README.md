# Mercurius iOS App

Native Swift/SwiftUI iOS app for the Mayo AI Literacy Club's Mercurius tutor. Replaces the React Native app in `/mobile`.

## Architecture

- `Mercurius/` — the thin `@main` app target. Composes the packages below and serves as the entry point.
- `Packages/DesignSystem/` — colors, typography, spacing, reusable UI primitives.
- `Packages/NetworkingKit/` — single `APIClient`, typed errors, session identity (Keychain-backed), response models.
- `Packages/AppFeature/` — composition root (`AppEnvironment`) plus high-level views that wire features together.
- Feature-specific packages (`ChatFeature`, `CurriculumFeature`, etc.) will be added as they're built.

The project file (`Mercurius.xcodeproj`) is **generated by `xcodegen` from `project.yml`** and is not checked in. Edit `project.yml`, then run `xcodegen` to regenerate.

## Local setup

```bash
# one-time
brew install xcodegen

# every time the project layout changes:
cd ios
xcodegen
open Mercurius.xcodeproj
```

## Running

- **Simulator:** ⌘R in Xcode.
- **Tests (all):** ⌘U in Xcode.
- **Tests (package only):** `swift test` inside `Packages/`.

## Bundle identifier

Currently `com.mayoailiteracy.mercurius.native` to avoid conflict with the React Native app's `com.mayoailiteracy.mercurius`. Swap to `com.mayoailiteracy.mercurius` when the React Native app is retired.

## Backend

Points to `https://mercurius-chatbot-production.up.railway.app` (production). For local dev, use `APIEnvironment.local` which targets `http://localhost:3000`.

## Standards

Every change should:

- Feel native (SwiftUI first, UIKit only when clearly better)
- Handle loading / empty / error / retry / offline states
- Support VoiceOver, Dynamic Type, Reduce Motion, dark mode
- Avoid main-thread blocking (use `async/await` with proper actors)
- Come with unit and/or UI tests
- Not log sensitive user content
- Not hard-code credentials
